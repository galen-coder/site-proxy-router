# 站点代理路由器 v2.3.1 发布日志

## 变更摘要

本次更新调整了临时路由与“不走代理列表”的优先级。临时路由现在是最高优先级，用于覆盖当前会话内的站点路由行为。

## 修复

- 修复当前域名命中“不走代理列表”后，弹窗中无法设置“临时直连”或“临时走指定代理”的问题。
- 修复 PAC 生成逻辑中“不走代理列表”优先于临时路由，导致临时代理实际不生效的问题。
- 修复路由状态预览中同一优先级关系的展示不一致问题。

## 行为变化

新的路由优先级为：

```text
1. 当前会话的临时路由
2. 不走代理列表
3. 已启用的分流规则（从上到下，第一条命中）
4. 未匹配网站出口：DIRECT 或默认代理及其回退链
```

这意味着：

- 命中“不走代理列表”的站点默认仍然直连；
- 但用户可以在插件弹窗或右键菜单中临时覆盖它；
- 临时路由只在当前会话内生效，浏览器重启、扩展重载或升级后会自动清除；
- 不走代理列表仍然优先于普通分流规则和默认出口。

## 文档与界面

- 更新弹窗提示文案，命中“不走代理列表”时会说明仍可设置临时路由。
- 更新设置页说明，明确“临时路由优先于例外列表，例外列表优先于普通分流规则”。
- 更新 README 中的路由优先级和版本说明。
